# TERM Schema — قالب المصطلح الموحّد

## الغرض

هذا الملف يحدّد **القالب الإلزامي** لتوثيق أي مصطلح في قاموس AUNAK. كل مصطلح معتمد مستقبلاً يُكتب وفق هذا الهيكل **دون استثناء**.

**هذا الملف قالب فقط — لا يحتوي على مصطلحات فعلية.**

---

**حالة الوثيقة:** Draft  
**الإصدار:** 0.1  
**تاريخ الإنشاء:** 2026-07-06

---

## كيفية استخدام القالب

1. انسخ القسم «قالب المصطلح» أدناه إلى ملف جديد داخل مجلد فرعي مخصص للمصطلحات المعتمدة (يُنشأ لاحقاً).
2. املأ كل حقل وفق `NAMING_RULES.md`.
3. مرّر المصطلح عبر `APPROVAL_PROCESS.md` قبل نشره.
4. بعد الاعتماد، حدّث حقل **الحالة** إلى `approved` وسجّل **تاريخ الاعتماد**.

## وصف الحقول

| الحقل | إلزامي | الوصف |
|-------|--------|-------|
| **TERM-ID** | نعم | معرف فريد ثابت؛ لا يتغيّر حتى عند تعديل الاسم |
| **الاسم العربي** | نعم | الاسم الوحيد المعتمد في الواجهات والوثائق العربية |
| **الاسم الإنجليزي** | نعم | الاسم الوحيد المعتمد في الكود والـ APIs والوثائق الإنجليزية |
| **التصنيف** | نعم | يُسهّل الفهرسة والبحث والربط بين الطبقات |
| **التعريف الرسمي** | نعم | المعنى المؤسسي — للبشر والوثائق |
| **التعريف التشغيلي** | نعم | المعنى التنفيذي — للمهندسين والأنظمة |
| **الكلمات المسموح بها** | نعم | صريحة حتى لو كانت «لا يوجد» |
| **الكلمات الممنوعة** | نعم | تمنع التسرّب اللغوي إلى النظام |
| **الاختصارات** | نعم | صريحة حتى لو كانت «لا يوجد» |
| **يستخدم في** | نعم | يربط المصطلح بالتنفيذ |
| **مرتبط بـ** | لا | يُفضَّل للمصطلحات ذات العلاقات |
| **الحالة** | نعم | lifecycle واضح |
| **الإصدار** | نعم | يتتبّع تطوّر التعريف |
| **تاريخ الاعتماد** | عند الاعتماد | سجل زمني رسمي |
| **ملاحظات** | لا | سياق مساعد دون تعريف بديل |

## قواعد القالب

- **لا يُنشر** أي ملف مصطلح بحالة `proposed` أو `under_review` كمرجع رسمي.
- **لا يُترك** حقل إلزامي فارغاً؛ استخدم «—» أو «لا يوجد» حيث ينطبق.
- **لا تُدرج** أمثلة مصطلحات حقيقية في هذا الملف — الأمثلة بين أقواس تعليقات للإرشاد فقط.

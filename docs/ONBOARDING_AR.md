## 1) ما هو عونك؟

منصة تشغيل لمراكز التربية الخاصة في سلطنة عُمان، بمسار يخدم المراكز ولجان الرقابة / وزارة التنمية الاجتماعية.

**التركيز الحالي (مهم):**  
الحوكمة والتشغيل — حضور معتمد، أهداف فردية (IEP)، أدلة تحقق، بروتوكول تقييم إجرائي، وصلاحيات حسب الدور.

**ليس التركيز الآن:** الوحدات المسرحية / الألعاب / مختبر الألحان (مخفية إلا بـ `?full=1`).

---

## 2) مسار المنتج

1. تسجيل بيانات المستفيد  
2. تقييم أولي / بروتوكول تقييم إجرائي  
3. بوابة التفعيل (اشتراك / كود)  
4. **البصمة الحيوية فقط بعد الاشتراك النشط**  
5. التشغيل اليومي: حضور → أهداف → أدلة → تقارير / رقابة  

الواجهة الافتراضية للإدارة/الأخصائي: **الحضور والأهداف الفردية**.

---

## 3) الأدوار

| الدور | ماذا يرى تقريباً |
|--------|-------------------|
| إدارة / سيادي | كامل المسار الجوهري + الصلاحيات |
| أخصائي / معلم تربية خاصة | حضور، أهداف، جلسات، تقييم |
| ولي أمر | تقارير / حضور معتمد (حسب التوكن) |
| رقابة وزارية | لوحات مُعمّاة (B2G) بدون بيانات شخصية ظاهرة |

## 4) Airtable — القاعدة المركزية

### جداول أساسية (أبقِها)

| جدول | الغرض |
|------|--------|
| `Centers` | سجل المراكز + نوع (خاص/حكومي) + نوبات الدوام |
| `Students` | المستفيدون + اشتراك + أهداف + تقييم + توكنات |
| `Specialists` | المعلمون والأخصائيون حسب التخصص والمركز |
| `Access Control` | رموز الدخول والصلاحيات |
| `Daily Sessions` | توثيق الجلسات المعتمد |
| `Session Periods` | جدول الحصص اليومية حسب التخصص والفترة |
| `Attendance Ledger` | حضور يومي معتمد (غير قابل للتلاعب الصامت) |
| `Goal Evidence` | أدلة تحقق أهداف الخطة الفردية |
| `Attendance Corrections` | طلبات تصحيح الحضور (الأصل يبقى) |

### قواعد التسمية

- أسماء الأعمدة: **إنجليزي `snake_case` فقط** (مثال: `student_name`)  
- لا تضع عربي أو `|` داخل اسم العمود  
- العربي يكون في **Description** أو في واجهة التطبيق  
- قيم الـ select صغيرة: `active`, `pending`, `morning`, `evening`, `day`…

### نوبات الدوام

- مركز **خاص:** `shift_model = dual` → `morning` + `evening`  
- مركز **حكومي:** `shift_model = single` → `day` فقط  

كل صف تشغيلي يجب أن يحمل `center_code` (وربط المركز).

### التخصصات المهنية (`specialty`)

`special_education` · `speech_language` · `occupational_therapy` · `physiotherapy` · `psychology` · `social_work` · `early_intervention`

## 5) التقنية

- **Frontend:** React + Vite  
- **نشر:** Vercel (`aunak.vercel.app`)  
- **بيانات:** Airtable عبر بروكسي السيرفر `/api/airtable`  
- **مهم:** مفتاح Airtable (`AIRTABLE_API_KEY` / PAT) على السيرفر فقط — **لا** تضعه في متغير يبدأ بـ `VITE_`

### ملفات يجب قراءتها

| ملف | لماذا |
|-----|--------|
| `docs/CENTRAL_BASE_SCHEMA.md` | مخطط القاعدة المركزية |
| `docs/VERCEL_FIX_TABLE_IDS_NOW.md` | قيم جداول Vercel الصحيحة |
| `docs/VERCEL_CENTRAL_BASE_ENV.md` | متغيرات البيئة للقطع للقاعدة الجديدة |
| `src/lib/centralAirtable.js` | IDs الجداول المعتمدة في الكود |
| `src/lib/airtableFields.js` | أسماء الأعمدة |
| `src/lib/hubNavConfig.js` | القائمة المختصرة vs الوحدات المسرحية |
| `.cursor/rules/aunak-sovereign-memory.mdc` | ذاكرة تشغيل سريعة للمشروع |

## 6) التشغيل المحلي

```bash
npm install
npm run dev
```

افتح: http://localhost:5173  

يحتاج `.env.local` (لا يُرفع على Git) وفيه على الأقل:

- `AIRTABLE_API_KEY` أو `VITE_AIRTABLE_PAT` للتطوير المحلي بحذر  
- `VITE_AIRTABLE_BASE_ID=appcjitgWsbvIebwf`  
- `VITE_USE_AIRTABLE_PROXY=true` (مُفضّل)

اسأل قائد الفريق عن ملف بيئة آمن — لا تنسخ أسراراً من الشات.

## 7) ماذا تعمل في أول أسبوع؟

1. افهم مسار الحضور → الأهداف → الأدلة → البروتوكول  
2. جرّب الموقع بعد الحصول على رمز دخول تجريبي من الإدارة  
3. اقرأ `CENTRAL_BASE_SCHEMA.md`  
4. لا تحذف جداول Airtable؛ إن لزم أرشف بالاسم `ARCHIVE_…`  
5. أي PR صغير وواضح؛ لا تخلط إصلاحات غير مرتبطة
